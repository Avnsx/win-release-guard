`0.3.5` stops the library's short-lived console helper processes from flashing
visible windows on Windows: a GUI host such as a PySide6 admin app that calls
`check_current_system(...)` previously saw two `powershell.exe` windows and one
`dism.exe` window pop up and steal focus on every check, and those windows are now
hidden. The release also folds in the earlier dashboard info-icon tooltip fix and
the fixed-date Pages freshness-fixture stabilization. No signed verdict,
`required_baseline_build` selection, `/api/v1` contract, Source Diagnostics
issue-sync rule, WUA-secondary behavior, or the 14-day baseline-update notice
window changed.

## Hidden Console Helper Windows

The library spawns short-lived console helpers to read local Windows state:
`powershell.exe` for `Get-CimInstance Win32_OperatingSystem` and `Get-WinEvent`,
`dism.exe` for `/Get-CurrentEdition` and `/Get-Packages`, and a re-exec of the
package itself (`python -m win11_release_guard.wua_probe`) for the secondary WUA
probe. Previously these had no window-hiding parameters, so when the parent was a
GUI process (for example a PySide6 admin app calling `check_current_system(...)`)
each child briefly popped a black console window and stole focus.

A new private helper, `win11_release_guard._subprocess_util.hidden_console_kwargs()`,
returns the Windows console-hiding `subprocess` keyword arguments and is merged
into every internal `subprocess.run` call:

- `creationflags` carrying `CREATE_NO_WINDOW` (`0x08000000`), resolved through
  `getattr(subprocess, "CREATE_NO_WINDOW", 0x08000000)` so the module imports
  cleanly off Windows.
- a `STARTUPINFO` with `STARTF_USESHOWWINDOW` set and `wShowWindow = SW_HIDE (0)`,
  which hides the window of any intermediate shell.

Both mechanisms are applied together (belt-and-suspenders) because Microsoft
documents each as a way to suppress a console child of a GUI parent. On
non-Windows platforms the helper returns a no-op pair
(`{"creationflags": 0, "startupinfo": None}`), so Linux/macOS and CI behavior is
unchanged.

This is a window-visibility change only. The commands, search criteria, timeouts,
encodings, parsed output, exit codes, and the resulting compliance verdict are
identical to `0.3.4`. The behavior is default-on with no opt-out flag, config
knob, or environment variable. Confirming that no window flashes on screen still
requires a real Windows GUI host; the suite verifies the hiding arguments are
passed through at the `subprocess.run` boundary without spawning real processes.

## Folded-In Fixes

`0.3.5` also publishes the earlier dashboard and fixture fixes that were staged
since `0.3.4`:

- The dashboard info-icon hover tooltips are restored. The tooltip bubble was
  `position: fixed` inside a `backdrop-filter` `<main>`, so it resolved against
  `<main>` and landed far below the fold; it is now `position: absolute`, anchored
  under its icon and contained within the viewport.
- Generated Pages freshness rendering is stabilized in fixed-date tests.
  `render_policy_index()` and `write_policy_outputs()` accept an optional
  render-age reference used only by tests and fixture helpers, while production
  output still computes freshness from the real current UTC time.

## Unchanged Verdict Boundaries

- Signed public policy JSON plus detached signature remains the compliance verdict authority.
- Local Windows labels, WUA, Panther/setup logs, DISM, Event Logs, and Source Diagnostics remain diagnostic evidence.
- 25H2 remains the existing-device broad-fleet target.
- 26H1 remains excluded for existing-device targeting.
- `/api/v1` policy, signature, and manifest aliases stay backward-compatible.
- Notice Source Diagnostic events remain dashboard-only and are not GitHub-Issue-syncable.
- The baseline-update notice is informational dashboard output only, visible for 14 days.
